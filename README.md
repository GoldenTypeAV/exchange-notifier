# Exchange Notifier

Телеграм-бот на Go, который подключается к приватному WebSocket биржи и присылает уведомления о торговых событиях (исполнение сделок) в указанный чат Telegram.

Сейчас поддерживается **Bybit** (приватные топики `order`, `execution`, `wallet`, `position`). Архитектура построена вокруг интерфейса `Exchange` + фабрики, поэтому добавление новой биржи сводится к реализации интерфейса.

## Возможности

- Подключение к приватному WS Bybit с HMAC-SHA256 аутентификацией.
- Поддержание соединения через периодический `ping` (keep-alive).
- Подписка на нужные приватные топики (настраивается в коде через `PrivateTopics`).
- Отправка форматированных уведомлений об исполнении ордеров в Telegram.
- Структурированное JSON-логирование (`log/slog`).
- Корректное завершение по `SIGINT` / `SIGTERM`.

## Требования

- Go 1.24+
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather)) и ID чата
- API-ключ Bybit с правами **только на чтение**

## Конфигурация

Скопируйте `.env.example` в `.env` и заполните значения:

```env
TELEGRAM_BOT_TOKEN=   # токен бота от @BotFather
TELEGRAM_CHAT_ID=     # числовой ID чата/канала для уведомлений
EXCHANGE=bybit        # пока поддерживается только bybit

BYBIT_API_KEY=
BYBIT_API_SECRET=
```

> **Важно:** `.env` добавлен в `.gitignore` и не должен попадать в репозиторий.
> Используйте API-ключ Bybit только с правами на чтение — приватный ключ
> для торговли или вывода средств этому боту не нужен.

## Запуск

### Локально

```bash
export $(grep -E '^[A-Za-z]' .env | xargs) && go run .
```

### Docker Compose

```bash
docker compose up -d --build
```

`docker-compose.yml` читает переменные из `.env`, ограничивает память (512 МБ),
настраивает ротацию логов и перезапуск (`restart: unless-stopped`).

## Структура проекта

```
.
├── main.go                       # точка входа: конфиг, бот, цикл обработки сообщений
└── internal/exchanges/
    ├── exchange.go               # интерфейс Exchange и PrivateTopics
    ├── factory.go                # фабрика New(name) -> Exchange
    └── bybit.go                  # реализация для Bybit (WS, auth, парсинг, форматирование)
```

## Как добавить новую биржу

1. Создайте файл `internal/exchanges/<name>.go` с типом, реализующим интерфейс `Exchange`.
2. Зарегистрируйте его в `factory.go` в `switch`.
3. Укажите `EXCHANGE=<name>` в `.env`.

## Статус

Проект в активной разработке. Реализована базовая отправка уведомлений об
исполнении сделок Bybit; обработчики `wallet` и `position` пока заглушки.
